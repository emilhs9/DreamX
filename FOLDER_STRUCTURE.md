# LaunchPad Folder Structure

```text
launchpad/
  client/
    index.html
    src/
      App.tsx
      Charts.tsx
      main.tsx
      styles.css
  server/
    app.js
    auth.js
    config.js
    deployer.js
    mailer.js
    migrate.js
    queue.js
    schema.sql
    security.js
    seed.js
    socketHub.js
    store.js
  migrations/
    001_initial_schema.sql
  docker/
    frontend-nginx.conf
  data/
    launchpad.json              # local fallback only
  deployments/
    <project-slug>/             # live project files
  uploads/
    avatars/
  tmp/
    builds/
      <deployment-id>/            # sandboxed build directories
  dist/
    client/                     # generated by npm run build
  node_modules/                 # generated by npm install
  .env.example
  .env.development
  .dockerignore
  ARCHITECTURE.md
  Caddyfile
  Dockerfile
  docker-compose.yml
  docker-compose.windows.yml
  ecosystem.config.js
  nginx.launchpad.conf
  package.json
  setup.sh
  server.js
  vite.config.ts
  tailwind.config.js
```

Production uses PostgreSQL through `DATABASE_URL`. When `DATABASE_URL` is empty, the
same API runs in local development with `data/launchpad.json` so the app can be tested
without a database server.

Key routes:

```text
/                         landing page
/login, /register         user auth cards
/dashboard, /deploy       protected user workspace
/dream                    separate admin login
/dream/*                  protected admin panel
/api/v1/*                 user/public REST API
/api/admin/*              separate admin REST API alias
/@project-slug/           local public project URL
```
